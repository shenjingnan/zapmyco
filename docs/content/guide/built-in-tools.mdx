ZapMyCo 的内置工具系统为 AI Agent 提供了与本地环境和用户交互的能力。大多数工具在 `zapmyco run <content>` 一次性执行模式下可用，而交互式工具（如 `ask_user`、`task_*`）在交互模式（`zapmyco` 无参数）下也可用。

## 工具速查

| 工具名称 | 功能 | 能力 |
|----------|------|------|
| [web_fetch](/guide/built-in/web_fetch) | 获取网页内容 | 将指定 URL 的网页转换为 Markdown，供 AI 分析 |
| [web_search](/guide/built-in/web_search) | 搜索网络 | 搜索实时信息（新闻、文档、趋势等） |
| [shell_exec](/guide/built-in/shell_exec) | 执行命令 | 在本地系统执行 shell 命令并返回输出 |
| [file_read](/guide/built-in/file_read) | 读取文件内容 | 按行号范围读取本地文件内容，支持分段查看 |
| [file_search](/guide/built-in/file_search) | 搜索文件内容 | 在本地文件系统中搜索文件内容，支持正则表达式 |
| [file_find](/guide/built-in/file_find) | 查找文件 | 按 glob 通配符模式（如 `**/*.rs`）在本地文件系统中查找文件 |
| [file_edit](/guide/built-in/file_edit) | 编辑文件内容 | 五种模式：line_range（推荐）、append、insert_after、批量编辑、旧模式 |
| [file_write](/guide/built-in/file_write) | 写入文件 | 创建新文件或完整覆盖已有文件，含预读检查和二进制检测 |
| [ask_user](/guide/built-in/ask_user) | 询问用户 | 向用户提出选项问题并获取回答，支持单选/多选和自定义输入 |
| [task_create](/guide/built-in/task_create) | 创建任务 | 创建新任务以跟踪复杂多步骤工作进度 |
| [task_get](/guide/built-in/task_get) | 查看任务 | 按 ID 获取单个任务的详情和状态 |
| [task_list](/guide/built-in/task_list) | 列出任务 | 列出当前所有任务及其状态概览 |
| [task_update](/guide/built-in/task_update) | 更新任务 | 更新任务的状态、字段和依赖关系 |

| 工具 | 说明 | 文档 |

AI Agent 会自动组合两者：先用 `web_search` 找到相关页面，再用 `web_fetch` 获取具体内容。

| 工具 | 说明 | 文档 |

| `file_read` | 读取文件内容，支持行号范围分段查看 | [查看详情](/guide/built-in/file_read) |

## 未来扩展

`ToolHandler` 枚举通过 `#[non_exhaustive]` 标记，预留了扩展能力，未来版本将持续新增工具数量。

| v0.32+ | `file_edit`, `file_write`, `ask_user` | 文件编辑、文件写入、用户询问 |

## 最佳实践

- **组合使用**：AI Agent 可自动组合多个工具，例如"搜索 Rust 最新版本 → 访问发布页面 → 总结变更内容"
- **`description` 参数**：在 prompt 中说明为什么需要执行某个命令，有助于 AI Agent 生成更有意义的描述，方便你判断是否授权
- **安全审慎**：只授权你确认过且安全的命令，对于删除、修改系统配置等危险操作应当慎重
- **大型页面限制**：`web_fetch` 对大文档页面输出上限为 100K 字符，AI 可能只能看到截断后的内容，复杂任务可考虑分步骤执行
