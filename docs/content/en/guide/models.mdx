title: Built-in Models
---

ZapMyCo comes with 22 models from 8 providers, all accessible via the Anthropic Messages API format.

You can reference model names directly in the `[llm.models]` section of `settings.toml`. You can combine models from different providers to achieve the best results. **Note: Each model's provider requires its corresponding API Key to be configured.**

```toml
[llm]

[llm.providers.deepseek]
apiKey = "sk-your-deepseek-key"

[llm.providers.anthropic]
apiKey = "sk-your-anthropic-key"

[llm.models]
default = "deepseek-v4-flash"
advanced = "claude-sonnet-4-6"
```

## OpenRouter

OpenRouter provides an Anthropic API-compatible `/v1/messages` endpoint, allowing you to access 400+ models from a single platform. Configure via custom `baseUrl`:

```toml
[llm.providers.openrouter]
apiKey = "sk-or-v1-xxxxx"
baseUrl = "https://openrouter.ai/api"

[llm.models]
default = "anthropic/claude-sonnet-4-6"
fast = "deepseek/deepseek-v4-flash"
```

> Model names use OpenRouter's namespace format, e.g., `anthropic/claude-sonnet-4-6`, `deepseek/deepseek-v4-flash`. For details, refer to [OpenRouter Model List](https://openrouter.ai/models).

## Local Models

The following inference frameworks natively support the Anthropic Messages API format and can be integrated via a custom `baseUrl`:

| Framework | Minimum Version | Default Port | Notes |
|-----------|:---------------:|:------------:|-------|
| **Ollama** | v0.14.0 | `:11434` | Easiest local deployment option |
| **llama.cpp** | Mainline supported | `:8080` | Native support for tool use, streaming, vision |
| **LM Studio** | v0.4.1 | `:1234` | GUI-based, out of the box |
| **LMDeploy** | Supported | `:23333` | Official Anthropic compatibility docs |
| **vLLM** | v0.11.1rc2+ | `:8000` | Warning: thinking tokens not yet supported |
| **SGLang** | Mainline supported | `:30000` | Fully compatible |
| **LocalAI** | v3.10.0+ | `:8080` | Built-in Anthropic compatibility layer |
| **Xinference** | v1.10.0+ | `:9997` | Endpoint prefix `/anthropic/v1/messages` |

The configuration method is uniform — just change `baseUrl` and the model name:

```toml
[llm]

[llm.providers.ollama]
apiKey = "ollama"
baseUrl = "http://localhost:11434/v1"

[llm.providers.llamacpp]
apiKey = "not-needed"
baseUrl = "http://localhost:8080/v1"

[llm.providers.lmstudio]
apiKey = "lmstudio"
baseUrl = "http://localhost:1234/v1"

[llm.models]
default = "qwen3"
fast = "llama3.2"
```

> Models from these frameworks are not in the built-in list — simply enter the name of your locally deployed model.

## Provider API Key via Environment Variables

While writing API Keys directly in `settings.toml` is simple, it poses a security risk (the config file might be accidentally committed to version control). Using environment variable references is recommended:

```toml
[llm.providers.deepseek]
apiKey = "${env.DEEPSEEK_API_KEY}"
```

At runtime, ZapMyCo automatically replaces `${env.DEEPSEEK_API_KEY}` with the value of the `DEEPSEEK_API_KEY` environment variable.

### Benefits

- **Security** — API Keys are not stored as plaintext in config files, avoiding accidental Git commits
- **Flexibility** — The same config file can use different keys across environments (development/CI/production)
- **Best Practice** — Aligns with industry standards and works naturally with Docker, CI/CD, and other toolchains

### When to Use

| Scenario | Recommended Approach |
|----------|---------------------|
| Personal local development | Either direct entry or environment variables |
| Shared config across multiple machines | Environment variables (config is shareable, each machine sets its own key) |
| CI/CD environment | Environment variables (injected via GitHub Secrets / CI variables) |
| Team-shared config files | Environment variables (avoid key leakage) |
| Quick trial and testing | Direct entry |

### How to Set

```bash
# Temporary (current terminal session only)
export DEEPSEEK_API_KEY="sk-your-key"

# Permanent (add to your shell config file)
echo 'export DEEPSEEK_API_KEY="sk-your-key"' >> ~/.zshrc

# Or use the zapmyco init wizard to auto-configure
zapmyco init
```
