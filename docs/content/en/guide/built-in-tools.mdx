ZapMyCo's built-in tool system provides AI Agents with the ability to interact with the local environment and the user. Most tools are available in `zapmyco run <content>` one-shot execution mode, while interactive tools (such as `ask_user`, `task_*`) are also available in interactive mode (`zapmyco` without arguments).

## Quick Reference

| Tool Name | Function | Capabilities |
|-----------|----------|--------------|
| [web_fetch](/built-in/web_fetch) | Fetch web content | Converts a specified URL's web page to Markdown for AI analysis |
| [web_search](/built-in/web_search) | Search the web | Searches for real-time information (news, docs, trends, etc.) |
| [shell_exec](/built-in/shell_exec) | Execute commands | Executes shell commands on the local system and returns the output |
| [file_read](/built-in/file_read) | Read file content | Reads local file content by line number range, supports segmented viewing |
| [file_search](/built-in/file_search) | Search file content | Searches for file content in the local filesystem, supports regex |
| [file_find](/built-in/file_find) | Find files | Finds files in the local filesystem by glob patterns (e.g., `**/*.rs`) |
| [file_edit](/built-in/file_edit) | Edit file content | Five modes: line_range (recommended), append, insert_after, batch edit, legacy mode |
| [file_write](/built-in/file_write) | Write files | Creates new files or fully overwrites existing files, with pre-read check and binary detection |
| [ask_user](/built-in/ask_user) | Ask the user | Presents choice-based questions to the user and gets answers, supports single/multi-select and custom input |
| [task_create](/built-in/task_create) | Create task | Creates a new task to track progress on complex multi-step work |
| [task_get](/built-in/task_get) | View task | Gets details and status of a single task by ID |
| [task_list](/built-in/task_list) | List tasks | Lists all current tasks and their status overview |
| [task_update](/built-in/task_update) | Update task | Updates a task's status, fields, and dependencies |
| [subagent](/built-in/subagent) | Sub-agent management | Creates independent sub-processes for concurrent task execution, supports spawn/poll/list/kill |
| [skill](/built-in/skill) | Skill management | Manages Skill workflow templates, supports list/load operations |

### Sub-agent Management

| Tool | Description | Docs |
|------|-------------|:----:|
| `subagent` | Creates independent sub-processes for concurrent sub-task execution, supports spawn/poll/list/kill operations | [View Details](/built-in/subagent) |

Sub-agents are marked with `--subagent` to prevent recursive spawning — sub-processes do not register the SubAgent tool. Output is automatically collapsed (>2KB summarized, >1MB truncated).

## Best Practices

- **Combined Use**: The AI Agent can automatically combine multiple tools, e.g., "search for the latest Rust version -> visit the release page -> summarize changes"
- **`description` parameter**: Explaining why a command needs to be executed in the prompt helps the AI Agent generate more meaningful descriptions, making it easier for you to decide whether to authorize
- **Safety First**: Only authorize commands you have verified as safe. Be cautious with dangerous operations such as deletions or system configuration modifications
- **Large Page Limit**: `web_fetch` has a 100K character output limit for large document pages. The AI may only see truncated content. For complex tasks, consider executing in steps
