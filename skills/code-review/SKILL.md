# 代码审查技能

对代码变更进行全面、专业的代码审查。

## 审查维度

### 1. 安全性
- SQL 注入、XSS、命令注入风险
- 敏感信息泄露（密钥、token）
- 权限校验是否完整

### 2. 正确性
- 边界条件处理
- 空值/null 检查
- 异常处理是否完善
- 类型安全

### 3. 性能
- 不必要的循环嵌套
- 未使用索引的查询
- 大文件/大数据处理方式

### 4. 可维护性
- 命名是否清晰
- 函数是否过于复杂
- 是否有适当的注释
- 是否遵循项目代码规范

### 5. 最佳实践
- 是否遵循项目 CLAUDE.md 中的约定
- 是否符合设计模式
- 是否有可复用的现有实现

## 执行步骤

1. 如果用户指定了范围，先理解变更的上下文
2. 运行 `git diff` 查看变更内容
3. 逐文件审查代码
4. 按优先级列出发现的问题：
   - P0: 必须修复（安全、数据丢失）
   - P1: 应该修复（逻辑错误、性能问题）
   - P2: 建议修复（代码风格、可读性）
5. 给出具体的改进建议（最好附带代码示例）

## 输出格式

```markdown
## Code Review 结果

### 总体评价
[1-2 句总体评价]

### 问题列表
- [P0] 问题描述 — 文件:行号 — 建议
- [P1] 问题描述 — 文件:行号 — 建议
- [P2] 问题描述 — 文件:行号 — 建议

### 亮点
[做得好的地方]
```
